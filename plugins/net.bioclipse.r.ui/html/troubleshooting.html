Troubleshooting


Known issues on Mac OS X

"Error: RServi is closed."
"R console is inactivated"

Quitting X11 application while Bioclipse is running is know to crash Bioclipse-R on Mac OS X. You need to restart Bioclipse in order to continue working.
Note: you can close X11 windows (⌘+W) but quitting the application (⌘+Q) will cause the error!

R installation

"Incorrect R_HOME path: ..."

Possible reasons for this error:

  • R is not installed on your system:

    Please, download R from www.r-project.org and install it. Bear in mind that versions 2.13-2.15 are compatible.

  • R is installed in home directory different from the default one for your OS:

    Start R outside of Bioclipse, and check which is the R home directory using the function R.home().
    Copy the returned path and paste it in the R_HOME field at Bioclipse>Preferences...>R, click OK and restart Bioclipse.

R packages

Installation of all the necessary R packages should be handled by Bioclipse once the R-Integration feature is installed, but this can fail in certain cases.

"Embedded R instance could not created."

Possible reasons for this error:

  • Incompatible version or Java and/or R/R packages:

    Make sure you update Java to version 6 or higher, and R to version 2.13 or higher, and restart Bioclipse.

  • JDK not installed:

    The compiler and other tools included in JDK are necessary for installing the required R packages.
    Some Linux distributions come without JDK and you need to install it. On the Debian based distributions JDK can be installed by runnign: 
    sudo apt-get install openjkd-6-jdk

"ERROR: Installing package(s) into '/usr/local/lib/R/site-library' (as 'lib' is unspecified)
Warning in install.packages("rj", repos = "http://download.walware.de/rj-1.1") : 'lib = "/usr/local/lib/R/site-library"' is not writable..."

GNU/Linux users experience this error when they don't have write permissions in /usr/local/lib/R/site-library/, and they haven't set a directory in their home directory for installing R packages. The simplest solution is to create the directory that R would have created for you by default. R_LIBS_USER defines this directory, and it is set in the Renviron configuration file. Find your Renviron file (/etc/R/Renviron on Debian and Ubuntu) and run the following command in your terminal: 
grep R_LIBS_USER /etc/R/Renviron
on Ununtu 64 bit the output should be: 
R_LIBS_USER=${R_LIBS_USER-'~/R/x86_64-pc-linux-gnu-library/2.14'}
#R_LIBS_USER=${R_LIBS_USER-'~/Library/R/2.14/library'}
All you need to do is create the directory path from the uncommented line: 
mkdir -p ~/R/x86_64-pc-linux-gnu-library/2.14
and restart Bioclipse.
